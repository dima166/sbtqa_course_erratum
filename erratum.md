# Список замечаний к внутреннему курсу по автоматизации тестирования

## Общие рекомендации

Предлагаются следующие рекомендации (замечания) в целом по курсу:

- **Следует выдерживать стиль небольших уроков**. Из положительных сторон такого подхода можно отметить: 
    - небольшие уроки легче проходить "на ходу", между делом; 
    - небольшие уроки снижают порог вхождения в обучение; 
    - такой подход соответствует особенностям учащихся из поколений Y и Z.

`_Сейчас есть небольшие уроки, а есть такие, над которыми нужно сидеть весь вечер. Курс неоднороден._`

- **Видео-уроки следует дополнять текстовым материалом**. Такой подход позволяет:  
    - быстро повторять пройденный материал; 
    - использовать пройденный курс в качестве справочного материала; 
    - знакомиться с материалом, даже если нет возможности посмотреть видео. 

`_Текстовые материалы также помогают структуировать, систематизировать пройденные уроки (особенность нашего мозга)._`

- **Следует побуждать учащегося к практическим действиям и дальнейшему изучению темы** `(активное вовлечение в процесс обучения)`. Например, можно сделать 
следующее:
    - в конце каждого учебного блока добавить рекомендованные книги, ссылки на ресурсы в сети + упомянуть, какое дальнейшее развитие может быть у идеи (ниже попробую показать);
    - можно попробовать сделать какой-нибудь интерактивный элемент, до поры до времени скрывающий результат прогона кода, и подзуживающий учащегося: 
"А у тебя так же выглядит прогон? :smile:" 

- **Следует использовать метод промежуточных проверок как способ закрепления материала**. В конце конце каждого учебного блока можно добавить небольшой 
проверочный тест (как показано в работах [Barbara Oakley](https://barbaraoakley.com/) это улучшает запоминание пройденного материала). 

`_Сейчас практическое задание в курсе стоит после большего объема пройденного материала_`.
